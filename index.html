Módulo 1: Anatomía de un Proyecto PHP Profesional

📌 Introducción

En el desarrollo académico, es común ver archivos index.php de 500 líneas que mezclan HTML, lógica de negocio y consultas SQL. En la industria, esto es inmantenible e inseguro. Este módulo establece las bases de una arquitectura escalable, centrada en la separación de responsabilidades y el punto único de entrada (Front Controller).

1.1 El Patrón Front Controller

En lugar de tener login.php, registro.php y perfil.php como archivos físicos accesibles, configuramos el servidor web para que toda petición pase por un único archivo: public/index.php. Este archivo actúa como un "despachador" (dispatcher).

// public/index.php
declare(strict_types=1);

// 1. Cargar configuración y autoloader
require_once __DIR__ . '/../config/bootstrap.php';

// 2. Enrutamiento básico (Simplificado para fines didácticos)
$uri = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH);
$method = $_SERVER['REQUEST_METHOD'];

// 3. Despacho al controlador correspondiente
if ($uri === '/usuarios' && $method === 'GET') {
    $controller = new App\Controllers\UsuarioController();
    $controller->index();
} else {
    http_response_code(404);
    echo "404 - Ruta no encontrada";
}

1.2 Autoloading Nativo (PSR-4 sin Composer)

Para evitar decenas de require_once, PHP ofrece spl_autoload_register. Esta función intercepta la instanciación de una clase y carga el archivo automáticamente basándose en su namespace.

// config/autoload.php
spl_autoload_register(function (string $class) {
    // Mapear el namespace "App\" a la carpeta "app/"
    $prefix = 'App\\';
    $base_dir = __DIR__ . '/../app/';

    // ¿La clase usa el namespace prefix?
    $len = strlen($prefix);
    if (strncmp($prefix, $class, $len) !== 0) {
        return; // No, salir
    }

    // Obtener el nombre de la clase relativo al prefix
    $relative_class = substr($class, $len);

    // Reemplazar el separador de namespace por el separador de directorio
    $file = $base_dir . str_replace('\\', '/', $relative_class) . '.php';

    // Si el archivo existe, cargarlo
    if (file_exists($file)) {
        require $file;
    }
});

1.3 Estructura de Directorios Segura

  • /public: Única carpeta expuesta a la web. Contiene index.php, CSS, JS e imágenes. El servidor web (Apache/Nginx) debe apuntar su "Document Root" aquí.
  • /app: Lógica de negocio. Inaccesible directamente vía HTTP. Contiene Controllers/, Models/, Views/.
  • /config: Archivos de configuración y el bootstrap.php que inicia la aplicación.
  • /storage: Fuera del directorio público. Aquí van los logs/ y los uploads/ de usuarios (evita que ejecuten scripts PHP subidos).

💡 Conclusiones y Consejos Profesionales

  • Regla de oro: Si un archivo PHP no necesita ser accedido directamente por una URL, no debe estar en la carpeta public/.
  • Namespaces: Usar namespace App\Controllers; en la parte superior de cada archivo es obligatorio para que el autoloader funcione. El nombre del archivo debe coincidir exactamente con el nombre de la clase (ej: UsuarioController.php).
  • Tipado estricto: Inicia siempre tus archivos con declare(strict_types=1); para forzar la coherencia de tipos de datos y prevenir bugs silenciosos.
