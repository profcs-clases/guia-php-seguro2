Módulo 5: Checklist de Despliegue y Recursos

📌 Introducción

El paso de desarrollo a producción es crítico. Un error de configuración puede exponer toda la base de datos. Esta página contiene un checklist imprimible y recursos para continuar tu formación como ingeniero de software.

✅ Checklist de Seguridad para Producción

📚 Glosario Técnico Esencial

  • Front Controller: Patrón de diseño donde un único script (ej: index.php) maneja todas las peticiones HTTP, centralizando la seguridad y el enrutamiento.
  • Session Fixation: Ataque donde un atacante fuerza a la víctima a usar un ID de sesión conocido. Se previene con session_regenerate_id(true) al autenticar.
  • Salting: Añadir una cadena aleatoria única a cada contraseña antes de hashearla. Previene que dos usuarios con la misma contraseña tengan el mismo hash y neutraliza los ataques con "Rainbow Tables".
  • Middleware: Código que se ejecuta en el "medio" de la petición HTTP, antes de llegar al controlador final, ideal para autenticación y logging.

ℹ️ Cultura Profesional (Informativo / No Evaluable)

Dominar estos conceptos te dará una ventaja competitiva masiva en tu primer empleo, aunque no sean requisito para aprobar esta cátedra:

  • Composer y Autoloading PSR-4: En la industria, no escribimos el spl_autoload_register a mano. Usamos composer init y composer dump-autoload.
  • Variables de Entorno (.env): Usa la librería vlucas/phpdotenv para cargar credenciales desde un archivo .env que nunca se sube a Git.
  • Git Workflow: Trabaja en ramas feature/nombre. Nunca hagas push directo a main. Usa Pull Requests para revisión de código.
  • Testing: Familiarízate con PHPUnit o Pest. Escribir pruebas unitarias para tus modelos y validadores es el sello de un ingeniero senior.
  • Xdebug: Instala esta extensión de PHP. Te permite poner "breakpoints" en tu código y ejecutar línea por línea desde tu IDE (VS Code / PhpStorm), en lugar de usar var_dump().
